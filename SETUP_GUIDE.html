❓ Your Questions Answered First — Read This!

Q: Can I use BOTH H2 (local) and PostgreSQL (online) at the same time?
YES — and your new code already does this automatically.
The application.properties is now smart:
• On your laptop → uses H2 file database automatically (no setup needed)
• On Railway (deployed) → uses PostgreSQL automatically (Railway sets this up)
You never touch this file again. It detects which one to use by itself.
Q: Will I lose my H2 data when I deploy online?
The H2 database only lives on your laptop. When you deploy to Railway, you get a brand new clean PostgreSQL database online. Users who register online are saved there permanently — they can log in from any device, any WiFi, anytime. Your local H2 is just for testing on your laptop.
Q: Do I still need IntelliJ open after deployment?
NO. After Railway deployment, the Spring Boot server runs 24/7 in the cloud. You only need IntelliJ for local testing on your laptop. The ESP32 will connect to Railway's URL instead of your laptop IP.

Phase 1 — IntelliJ IDEA

☕ Set Up Spring Boot Backend

Open the project in IntelliJ

  1. Open IntelliJ IDEA
  2. Click "Open" on the welcome screen (NOT "New Project")
  3. Navigate to: final-project → Backend_Java_Spring → system → system
  4. You must see pom.xml inside this folder — that's the right folder
  5. Click OK / Select Folder

✅ How to know you opened the right folder
In the left panel (Project tree), you should see: src folder, pom.xml, mvnw all at the same level. If you only see another folder inside, you went one level too shallow.

Set Java 21 SDK

  1. Go to File → Project Structure (shortcut: Ctrl+Alt+Shift+S)
  2. Click Project on the left side
  3. Look at SDK dropdown — if it's empty or wrong version:
  4. Click the dropdown → Download JDK
  5. Select version 21, vendor: Eclipse Temurin (or any)
  6. Click Download → wait → click Apply → OK
3

Load Maven (pom.xml) — downloads all dependencies

IntelliJ may show a notification at the top or bottom. Look for any of these:

  • "Maven build script found" → click the blue link
  • A floating popup → click "Load Maven Project"

If no popup appears:

  1. Right-click on pom.xml in the left panel
  2. Click Maven → Reload Project
📦 What pom.xml does — explained simply
pom.xml is like a shopping list for your project. It tells Maven (a download manager) which libraries to download. The first time it runs, it downloads everything from the internet (takes 2–5 minutes). After that it's cached locally and runs instantly.

Your pom.xml includes:
• spring-boot-starter-web → makes it a web API server
• spring-boot-starter-data-jpa → connects to database
• spring-boot-starter-security → BCrypt login protection
• spring-boot-starter-mail → Gmail SMTP for OTP/alerts
• h2 → local database (for your laptop)
• postgresql → online database (for Railway deployment)
• lombok → removes boilerplate (@Getter, @Setter)
• twilio → SMS alerts
⚠️ If you see red errors after loading Maven
This means dependencies didn't download. Fix: Go to File → Settings → Build, Execution → Maven → make sure "Always update snapshots" is checked. Then right-click pom.xml → Maven → Reload Project again. Make sure you have internet!
4

Enable annotation processing (for Lombok)

  1. Go to File → Settings (shortcut: Ctrl+Alt+S)
  2. Search for "annotation" in the search bar
  3. Click Build, Execution → Compiler → Annotation Processors
  4. Check ✅ "Enable annotation processing"
  5. Click Apply → OK
⚠️ Why this matters
Lombok uses annotation processing to generate @Getter, @Setter, @NoArgsConstructor etc. at compile time. Without this checked, IntelliJ shows red errors on all your model files even though the code is correct.
5

Run the Spring Boot application

  1. In the left panel, find: src/main/java/com/tank/system/TankSystemApplication.java
  2. Double-click to open it
  3. Click the green ▶ play button on the left margin next to public static void main
  4. OR click the green ▶ button at the very top toolbar of IntelliJ
  5. Wait 20–40 seconds for it to start

✅ Success — you will see this in the Console panel at the bottom:

Test it works: Open Chrome → go to http://localhost:8080/h2-console

Set up ESP32 in VS Code (PlatformIO)

  1. In VS Code: File → Open Folder
  2. Select the root final-project folder (the one with main.cpp + platformio.ini)
  3. Open main.cpp
  4. Find and fill in these 3 lines:
// LINE 5 — your WiFi name: const char* ssid = "YOUR_WIFI_NAME_HERE"; // LINE 6 — your WiFi password: const char* password = "YOUR_WIFI_PASSWORD_HERE"; // LINE 17 — your laptop's IP address on the WiFi: const char* serverUrl = "http://192.168.1.5:8080/api/tank/sync";
📡 How to find your laptop IP address
1. Press Windows + R → type cmd → press Enter
2. Type ipconfig → press Enter
3. Look for Wireless LAN adapter WiFi → IPv4 Address
4. It looks like 192.168.1.5 or 192.168.100.xx
5. Use that in the serverUrl above

⚠️ Your ESP32 and laptop MUST be on the same WiFi network!

Upload to ESP32:

  1. Connect ESP32 to laptop via USB cable
  2. At the bottom of VS Code, click the → Upload arrow button
  3. Wait for compilation and upload (about 30–60 seconds)
  4. Click the plug icon (Serial Monitor) → set baud to 115200
  5. You should see: Sync OK | Dist: 24.5cm | Level: 67.2% | Cmd: AUTO_OFF

Phase 4 — Online Deployment

🚀 Deploy to Railway + Netlify (Free)

Do this when you want accounts to be saved permanently online and accessible from any device

11

Push your code to GitHub

  1. Go to github.com → Sign up for a free account (or log in)
  2. Click the + icon → New Repository
  3. Name it: water-tank-system → set to Public → click Create
  4. Open Command Prompt on your laptop
  5. Navigate to your project folder:
cd Desktop\final-project\Backend_Java_Spring\system\system

Then run these commands one by one:

git init git add . git commit -m "Initial commit - Water Tank System" git branch -M main git remote add origin https://github.com/YOUR_USERNAME/water-tank-system.git git push -u origin main
⚠️ Replace YOUR_USERNAME
Change YOUR_USERNAME to your actual GitHub username. It will ask for your GitHub email and a token (not your password — GitHub no longer accepts passwords in terminal). To get a token: GitHub → Settings → Developer Settings → Personal Access Tokens → Generate new token → select "repo" → copy the token and use it as the password.
12

Deploy Backend on Railway (free PostgreSQL database online)

  1. Go to railway.app → Sign up free with GitHub
  2. Click New Project → Deploy from GitHub Repo → select water-tank-system
  3. Railway detects Spring Boot → starts building automatically
  4. Click + New → Database → Add PostgreSQL
  5. Railway automatically creates the database and sets DATABASE_URL for you
  6. Wait 3–5 minutes → you get a URL like: https://water-tank-xxxx.up.railway.app
✅ Test your backend is live
Open browser → go to: https://your-railway-url.up.railway.app/api/admin/stats
If you see JSON data → your backend is live and database is connected! 🎉
💡 How the database switch works automatically
Your application.properties has this line:
spring.datasource.url=${DATABASE_URL:jdbc:h2:file:./data/tank_db}

This means:
• If DATABASE_URL environment variable exists → use it (Railway sets this) → PostgreSQL
• If it doesn't exist (your laptop) → use H2 file

You never change application.properties. It just works.
13

Deploy Frontend on Netlify (free website hosting)

  1. Go to netlify.com → Sign up free (use GitHub login)
  2. On the dashboard: drag and drop your Frontend_Standalone folder onto the page
  3. Done! You get a URL like: https://water-tank-system.netlify.app
  4. Do the same for Admin Dashboard: drag and drop the Admin_Dashboard folder → get another URL
14

Update the Railway URL — only 1 line in each file

Now that you have your Railway URL, update it in 3 files:

File 1: Frontend_Standalone/dashboard.js (line 6)

// Change this: const RAILWAY_URL = "http://localhost:8080"; // To this (your actual Railway URL): const RAILWAY_URL = "https://water-tank-xxxx.up.railway.app";

File 2: Frontend_Standalone/app.js (line 8)

const RAILWAY_URL = "https://water-tank-xxxx.up.railway.app";

File 3: Admin_Dashboard/admin.js (line 3)

const RAILWAY_URL = "https://water-tank-xxxx.up.railway.app";

After changing, redeploy to Netlify by dragging the folder again.

15

Update ESP32 to use Railway URL

Open main.cpp → change line 17:

// Change this: const char* serverUrl = "http://192.168.1.5:8080/api/tank/sync"; // To this (your Railway URL): const char* serverUrl = "https://water-tank-xxxx.up.railway.app/api/tank/sync";

Then re-upload to ESP32 via PlatformIO → Upload button.

✅ After this, your prototype works on ANY WiFi anywhere
The ESP32 no longer needs to be on the same WiFi as your laptop. It just needs any internet connection and it syncs directly to Railway 24/7.
Troubleshooting

🔧 Common Errors and How to Fix Them

!

Error fixes

ErrorFix
Port 8080 already in useIntelliJ is already running. Click the red ■ Stop button in the Run panel at the bottom, then run again.
Red errors on model files in IntelliJFile → Settings → Compiler → Annotation Processors → check "Enable annotation processing" → Apply.
CORS error in browser consoleYour VS Code live server port changed. Open SecurityConfig.java → add your current port (5500 or 5501) to allowedOrigins. Both are already there in your new code.
QR scanner not workingAllow camera in browser: Chrome → Settings → Privacy → Site Settings → Camera → Allow for 127.0.0.1. After deployment on HTTPS it works on all phones.
ESP32 shows "HTTP Sync Failed -1"Your laptop IP changed. Run ipconfig again, get new IP, update main.cpp line 17, re-upload to ESP32.
Maven not downloading (red pom.xml)Check internet. Then: right-click pom.xml → Maven → Reload Project.
Railway build failsMake sure you pushed the folder that contains pom.xml (not a parent folder). Check Railway logs for the specific error.
Netlify site shows blank pageMake sure RAILWAY_URL in dashboard.js and app.js is set to your Railway URL, not localhost.
