Deploy Online (GitHub + Railway + Netlify) + Prototype Wiring (ESP32 + HC-SR04 + Pump)

❓ Your Questions — Answered Simply

Do I lose my H2 database when I deploy online? Can I use BOTH local and online?
YES you can use both — and your code already supports this automatically.

• H2 (local) = file on your laptop. Used when IntelliJ is running. For testing only.
• PostgreSQL (Railway) = database in the cloud. Used when deployed online. Permanent, accessible anywhere.

They are separate databases — not synced. What you register locally stays local. What you register online stays online permanently.

Your application.properties auto-detects which one to use:
→ On laptop: uses H2 automatically (no setup)
→ On Railway: uses PostgreSQL automatically (Railway sets it up)
Will the ESP32 prototype still connect after deploying online?
YES — 100% compatible. It gets BETTER after deployment.

Right now ESP32 sends data to your laptop IP (e.g. 192.168.1.5:8080) — needs same WiFi.
After Railway deployment, ESP32 sends to your Railway URL — works on any WiFi anywhere.
The ESP32 just needs internet. It doesn't care if the server is your laptop or a cloud server.
Will the prototype physically connect with the website system?
YES — this is exactly what your system is designed to do:

HC-SR04 sensor → measures distance to water → ESP32 calculates water level %
→ ESP32 sends level to Spring Boot (every 5 seconds)
→ Spring Boot decides: pump ON or OFF based on thresholds
→ ESP32 reads the decision → activates relay → pump turns ON or OFF
→ Your website dashboard shows live water level in real time
Do I need 2 tanks for the demo?
YES — for a proper demo you need 2 small containers:
• Tank 1 (main tank) — the one with water. HC-SR04 sits on top and measures water level.
• Tank 2 (reservoir/source) — smaller container below or beside it. Pump is submerged here and pumps water UP to Tank 1.

This shows the full auto-refill cycle: level drops → pump turns on → water fills back up → pump turns off.

🚂 Railway — Deploy Backend + PostgreSQL Database

Free tier. Hosts your Spring Boot server + creates PostgreSQL database online.

5

Create Railway Account

  1. Go to railway.app
  2. Click Login with GitHub (use the same GitHub account you just made)
  3. Authorize Railway to access GitHub
  4. You're in Railway dashboard
6

Deploy your Spring Boot from GitHub

  1. Click New Project
  2. Click Deploy from GitHub repo
  3. Find and select water-tank-system
  4. Railway auto-detects Spring Boot → starts building automatically
  5. Wait 3–5 minutes. You'll see the build logs scrolling.
⚠️ If build fails with "BUILD FAILED"
Check the logs — usually it's because Railway can't find pom.xml. Fix: In Railway → your service → Settings → look for "Root Directory" → set it to the folder path where pom.xml is located.
7

Add PostgreSQL Database (online database)

  1. Inside your Railway project, click + New (or the + button)
  2. Click Database → click Add PostgreSQL
  3. Railway creates the database automatically in about 30 seconds
  4. Railway also automatically sets the DATABASE_URL environment variable — your Spring Boot picks this up automatically
  5. You don't need to do anything else for the database
✅ How the auto-detection works (already in your code)
Your application.properties has this line:
spring.datasource.url=${DATABASE_URL:jdbc:h2:file:./data/tank_db}

This means: if DATABASE_URL exists (Railway sets it) → use PostgreSQL online
If not (your laptop) → use H2 local file
Zero changes needed. It just works.

  3. Look for Domains section → click Generate Domain
  4. You get a URL like: water-tank-xxxx.up.railway.app
  5. Copy this URL — you need it in the next steps
🧪 Test your backend is live

PHASE 5

🔧 Prototype Wiring — ESP32 + HC-SR04 + Pump

How to physically connect all your components together

13

Understand your components and what each does

🧠 ESP32 DevKit V1
The brain. Runs your main.cpp code. Reads sensor, controls pump, sends data to server via WiFi.
Powers from USB power bank (5V)
📡 HC-SR04 Ultrasonic Sensor
Measures distance from sensor to water surface. Mounted at the TOP of Tank 1 pointing down.
VCC=5V, GND, TRIG=GPIO5, ECHO=GPIO18
💧 12V Submersible Pump
Submerged in Tank 2 (source). Pumps water up to Tank 1 when level is low. Controlled by TIP120.
Controlled via TIP120 transistor from GPIO19
⚡ TIP120 NPN Transistor
Acts as a switch between ESP32 (3.3V signal) and the pump (12V). ESP32 cannot power pump directly.
Base=GPIO19 via 1kΩ resistor, Collector=pump(-), Emitter=GND
🔋 Power Bank (5V)
Powers the ESP32 via Micro USB. Also powers HC-SR04 via 5V pin from ESP32.
Connect to ESP32 Micro USB port
🔌 12V Power Source for Pump
Separate 12V supply for the pump. Can be a 12V adapter or 8x AA batteries. SEPARATE from ESP32 power.
12V(+) → pump(+), 12V(-) → TIP120 emitter → GND

2-Tank Setup Explanation

🏗️ Physical Layout — Side View
[HC-SR04 sensor on lid] ↕ measures distance ┌─────────────────┐ ← Tank 1 (MAIN TANK) │ WATER LEVEL │ Monitor this tank's level │ ~~~~~~~~~~~~~ │ Pump fills this from Tank 2 │ │ └─────────────────┘ │ (water flows via tube/pipe when pump ON) │ ┌──────────────┐ ← Tank 2 (RESERVOIR / SOURCE) │ 💧 water │ Smaller container │ [====PUMP===]│ Pump submerged here └──────────────┘ ESP32 + breadboard + TIP120 mounted on Sintra board nearby
💡 Why 2 tanks?
Tank 1 = the tank being monitored. Water level drops as it's "used" (you can manually drain it to simulate usage).
Tank 2 = the water source. Pump sits here and pushes water up a tube into Tank 1.
When Tank 1 level drops below your threshold → ESP32 gets AUTO_ON from server → TIP120 switches on → pump activates → water flows from Tank 2 to Tank 1 → level rises → ESP32 gets AUTO_OFF → pump stops.

For your small demo: use plastic food containers or bottles. Even 1-liter bottles work!

Complete Wiring Diagram — Step by Step

Part A — HC-SR04 Ultrasonic Sensor to ESP32

HC-SR04 PinConnect toWire Color (suggestion)Why
VCCESP32 VIN or 5V pin RedPower for sensor (needs 5V)
GNDESP32 GND BlackGround
TRIGESP32 GPIO 5 GreenTrigger pulse — ESP32 sends signal to sensor
ECHOESP32 GPIO 18 BlueEcho — sensor returns pulse to ESP32
⚠️ HC-SR04 ECHO pin is 5V — ESP32 is 3.3V!
The ESP32 GPIO pins are 3.3V tolerant only. The HC-SR04 ECHO pin outputs 5V.
Simple fix: Use a voltage divider between ECHO and GPIO18:
ECHO → 1kΩ resistor → GPIO18 → 2kΩ resistor → GND
OR use a logic level converter module (cheaper than a damaged ESP32!)
Many builders skip this and it works — but long-term it can damage the ESP32 GPIO.

Part B — TIP120 Transistor for Pump Control

📌 TIP120 has 3 legs — from left to right when facing flat side: Base, Collector, Emitter
TIP120 LegConnect toNotes
Base (B)ESP32 GPIO 19 via 1kΩ resistorControl signal from ESP32. The resistor protects GPIO19.
Collector (C)Pump negative (-) wireSwitch side — current flows through here to pump
Emitter (E)Common GNDMust connect to SAME ground as 12V supply and ESP32

Part C — Pump Power Connections

WireConnect toNotes
Pump positive (+)12V power supply positive (+)Direct connection — always has power when supply is on
Pump negative (-)TIP120 CollectorTIP120 opens/closes this circuit based on GPIO19
12V supply negative (−)TIP120 Emitter AND ESP32 GNDAll grounds must be connected together (common GND)
⚠️ Add a Flyback Diode across the pump
Pumps are motors — they create voltage spikes when turned off that can damage the TIP120.
Connect a 1N4007 diode across the pump: Cathode (stripe) to pump(+), Anode to pump(−).
This diode costs ₱5 and saves your transistor and ESP32.

Physical Assembly on Sintra Board

  1. Sintra board — lay it flat, this is your base. Attach everything to it.
  2. Tank 1 (main tank) — place on the board. Cut a small hole in the lid for the HC-SR04 sensor. Mount HC-SR04 inside the lid pointing straight down at the water.
  3. Tank 2 (reservoir) — place beside or below Tank 1. Submerge the pump inside. Run a tube from pump outlet up into Tank 1.
  4. Breadboard — mount with double-sided tape or screws on the Sintra board. Place TIP120, resistors, diode here.
  5. ESP32 — place on the breadboard or beside it. Connect via jumper wires.
  6. Power bank — connect Micro USB cable from power bank to ESP32.
  7. Project box / acrylic case — optionally cover the electronics to make it look clean for demo.
💡 Pro tip for neat demo
Use cable ties or tape to organize all wires. Label each wire with a small piece of tape and a pen. Judges/professors appreciate neat, organized prototypes as much as working ones.
